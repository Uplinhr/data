# Data – Análisis y Métricas  

El componente de **Data & Analytics** en este proyecto tuvo un papel secundario frente a los ejes principales:  
- Desarrollo web (landing page y funcionalidades asociadas).  
- Automatizaciones de procesos con Make y systeme.io.  
- Gestión de leads, clientes y comunicaciones.  

---

## Contexto  

El proyecto nunca contó con volúmenes significativos de datos para análisis profundo.  
Esto implicó que el área de data quedara **relegada** en esta etapa inicial, funcionando más como un soporte básico que como un pilar del proyecto.  

Aun así, se estableció una **estructura inicial aceptable** para recopilar información de usuarios y preparar el terreno para futuras expansiones.  

---

## Alcance actual  

- El foco de los datos recopilados está en el **flujo de usuarios de la landing page**.  
- Se utilizó **Google Analytics 4 (GA4)** para recolectar métricas de navegación.  
- Se construyeron tableros iniciales en **Google Looker Studio** para visualizar embudos de conversión y eventos clave.  
- La información gestionada representa solo una **pequeña fracción del proyecto**, en comparación con los módulos de desarrollo web, automatización y gestión.  

---

## Potencial de escalabilidad  

La arquitectura planteada permite crecer en caso de que aumente la base de usuarios o se amplíen los servicios:  
- Integración con **CRM** para enriquecer datos de GA4.  
- Almacenamiento en un **data warehouse** (ej. BigQuery o Microsoft Fabric) para histórico y análisis avanzado.  
- Dashboards más completos en **Looker Studio** o **Power BI**.  

---

## Conclusión  

Aunque el análisis de datos no fue el centro de este proyecto, se construyó una **base inicial sólida** que permitirá escalar en el futuro.  
Actualmente, el área de data sirve como **apoyo al monitoreo de la landing page**, pero está preparada para evolucionar hacia un esquema más robusto conforme el proyecto crezca.
